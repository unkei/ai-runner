# 直進弾と地面固定ゲート 実装プラン

## 目的

プレイテストのフィードバックに基づき、味方が撃った弾と「x2」などのゲート表示の前進表現を修正する。

- 味方弾は発射時に決めた方向へ直進し、発射後に敵を追って曲がらない。
- ゲートは自律的にプレイヤー側へ移動せず、コース上の位置に固定する。
- プレイヤーの前進距離が増えることで、固定されたゲートが相対的に近づいて見えるようにする。

この文書は実装方針のみを定義する。ゲーム本体は、別途明示的に実装を依頼されたときに変更する。

## 現状と変更後の挙動

### 味方弾

現状:

- `AllyBullet` は `targetEnemyId` を保持する。
- 各 tick で対象敵の現在位置を取得し、`moveToward` でその位置へ向きを変えながら進む。
- 対象敵が先に消えると、飛んでいる弾も即座に削除される。

変更後:

- 発射時の味方位置と対象敵位置から正規化した方向を一度だけ計算し、弾に `velocityX` / `velocityY` として保持する。
- 発射後は `x += velocityX * dt`、`y += velocityY * dt` だけで直進させる。敵の移動や消滅によって方向を変えない。
- 弾は特定の対象へ吸着させず、飛行経路上で最初に接触した生存敵へダメージを与える。
- 対象が先に倒されても弾は直進を続け、敵へ命中するか、有効範囲外へ出るまで残る。

### ゲート

現状:

- 各ゲートは画面奥を表す `y: -0.08` と `speed` を持つ。
- 各 tick で `gate.y += gate.speed * dt` と更新されるため、プレイヤーの前進とは独立してゲート自身が手前へ移動する。
- 描画、左右選択、通過判定はいずれも可変の `gate.y` に依存する。

変更後:

- 各ゲートはコース上の固定絶対座標 `worldDistance` を持つ。同じペアは同じ値を共有する。
- ゲートの相対奥行きは `worldDistance - state.distance` から毎 frame 導出し、投影用の `y` に変換する。
- `worldDistance` 自体は tick で変更しない。`state.distance` が増えた分だけ相対距離が減り、ゲートが近づいて見える。
- プレイヤーがゲート距離を横切った時点で、現在の `playerX` により左右どちらを取得したか決める。
- 通過済みのペアは取得処理後に削除する。見える範囲より後方へ抜けた未処理ゲートも残さない。

## スコープ

### 対象

- 味方弾の発射時方向、直進移動、敵との命中判定、寿命管理。
- 全ゲート表示（`+N`、`-N`、`xN` / `*N`、`/N`）のコース固定座標と相対投影。
- ゲートの生成位置、描画、取得判定。
- 上記仕様に関する決定論的テスト、README のゲーム説明。

### 対象外

- 敵の弾や矢の追尾仕様。今回の「打った球」は味方が撃つ弾へのフィードバックとして扱う。
- 味方の発射前の索敵、発射間隔、銃が敵方向を向く表現。
- 敵の移動、敵スポーン、左右操作、ゲートの演算内容や出現確率。
- コース上の敵を絶対距離座標へ移す全面的なワールド座標系の再設計。

## 状態モデルと処理変更

### `AllyBullet`

```text
AllyBullet {
  id: number
  sourceAllyId: number
  x: number
  y: number
  velocityX: number
  velocityY: number
  damage: number
  remainingLifetime: number
}
```

- `targetEnemyId` と追尾用 `speed` を正本から外す。
- 発射時にゼロ距離を安全に扱い、正面方向など決定論的な既定方向を使う。
- 高速弾が大きな `deltaSeconds` で敵を飛び越えないよう、tick 前後の線分と敵の当たり円の交差を使う。
- 1発が同時に複数敵へ当たる場合は、弾の進行方向に沿って最初の交点を採用し、同距離なら敵IDが小さい方を採用する。
- 命中した弾は削除する。未命中弾は寿命を減らし、寿命切れまたはコース外で削除する。

### `Gate`

```text
Gate {
  id: number
  side: "left" | "right"
  x: number
  worldDistance: number
  operation: string
  value: number
}
```

- `y` と `speed` を状態から外す。
- 生成時に `state.distance + GATE_LOOKAHEAD_DISTANCE` を基準として `worldDistance` を決める。
- 同時に複数ペアが存在しても識別が安定するよう、距離の丸め値ではなく明示的な `pairId` を追加するか、ペア共通の生成IDを使用する。
- 描画専用 helper が相対距離を投影用 `y` と scale に変換し、遠方からプレイヤー位置まで単調に近づく値を返す。
- 接触判定では前 frame の `distance` と更新後の `distance` の間に `worldDistance` が含まれるかを判定できるよう、tick 内で通過前距離を保持するか、許容幅を含む未通過判定を使う。

## レンダリング変更

- `drawProjectiles` は弾の現在座標だけを描画し、弾道の向きが分かるよう `velocityX` / `velocityY` を Canvas 上の方向へ変換して線分を描く。
- `drawGates` は各ペアの `worldDistance - state.distance` からその frame の投影位置を導出する。
- ゲートのバー、中央線、演算表示を同じ投影値で描き、`x2` の文字だけが独立して近づくように見えない状態を維持する。
- 遠方、画面中央付近、通過直前の順に表示サイズが単調に大きくなることを確認する。

## 決定論的テスト

`src/tests.js` に少なくとも次を追加または更新する。

- 発射時に弾の速度ベクトルが対象方向へ設定される。
- 発射後に対象敵が横移動しても、弾の速度ベクトルと直線軌道が変わらない。
- 元の対象敵が消えても、弾が即時削除されず直進を続ける。
- 直線経路上の敵へ命中し、経路外へ動いた元対象には曲がって命中しない。
- 1 tick の移動距離が敵の直径より大きくても線分判定で命中する。
- 同じ弾道上に複数敵がいる場合、最初の敵だけが被弾する。同距離の場合は敵IDで結果が安定する。
- 寿命切れまたはコース外の弾が削除される。
- 生成されたゲートペアが同じ固定 `worldDistance` とペアIDを持つ。
- `state.distance` が変わらない更新ではゲートの `worldDistance` が変わらない。
- `state.distance` の増加により相対距離だけが減り、投影位置が手前へ進む。
- 大きな `deltaSeconds` でゲート位置を横切っても一度だけ取得される。
- 左右選択が通過時の `playerX` に従い、選ばれなかった片側も同時に削除される。
- リセット後に弾とゲート状態が初期化される。

## ブラウザ検証

- `npm test` と `tests.html` の全テストが成功する。
- デスクトップ幅で、敵が横へ移動した後も発射済み弾が曲がらず直進することを目視確認する。
- 発射後に対象が別の弾で倒されても、残った弾が不自然に消えたり別の敵へ曲がったりしないことを確認する。
- ゲート出現中に観察し、ゲート全体が地面上の同じ位置に固定され、コースの前進表現に合わせて近づくことを確認する。
- 左右操作で `x2` などを通過し、演算が一度だけ適用されることを確認する。
- 低FPS相当の大きな frame 間隔でも弾とゲートを飛び越さないことを確認する。
- 1280x720 と 390x844 で、コンソールエラーと横方向オーバーフローがないことを確認する。

## 受け入れ条件

- 味方弾は発射後に進行方向を変更しない。
- 敵が移動または消滅しても、飛行中の弾は既定の直線軌道を維持する。
- 弾の見た目と命中判定が同じ直線軌道に一致する。
- ゲートの絶対位置は生成後に変化せず、プレイヤーの `distance` 増加だけで相対的に接近する。
- ゲート通過時の左右選択と人数演算が従来どおり正しく、一度だけ適用される。
- 自動テストと指定ブラウザ検証が成功し、既存の戦闘・操作・再スタートに回帰がない。

## リスクと前提

- ユーザーの「打った球」は味方弾を指すと解釈する。敵の矢も直進へ統一する場合は別スコープとする。
- 現在は敵と味方が画面内の正規化座標を使い、ゲートだけが絶対距離を持つ。投影変換の境界を明示しないと描画と接触判定がずれるため、変換 helper を一箇所へ集約する。
- 固定弾道では動く敵への命中率が下がる。これは要求された挙動だが、ゲームバランス調整は本変更に含めない。
- 低FPS時には点同士の距離判定だけでは弾やゲートを飛び越すため、線分交差と距離横断判定を必須とする。
