# 棒人間ごとの独立戦闘 実装プラン

## 目的

現在の人数カウンター単位の群衆戦闘を、味方・敵の棒人間がそれぞれ独立して行動する戦闘へ変更する。

- 各味方は生存中の敵を自動で見つけ、個別に狙って銃を撃つ。
- 射撃しても味方自身は消費されず、1体の味方が時間をかけて複数の敵を倒せる。
- 各敵は独立して移動、索敵、攻撃する。
- 敵の攻撃が命中すると、対象の味方がダメージを受ける。
- 敵と味方が直接ぶつかった場合は、その組み合わせの2体が相互に消える。

この文書は実装方針のみを定義する。ゲーム本体は別途明示的に実装を依頼されたときに変更する。

## 現状

- 味方は `squadSize` という人数だけで管理され、個体ごとのID、HP、射撃状態、対象を持たない。
- 敵は `count` を持つ敵グループで、グループ単位に移動・接触解決する。
- 接触時は味方人数と敵人数を一括で相殺する。
- 敵グループは固定された横位置を直進するため、個々の敵が別々の対象へ動くことはない。
- 敵側の攻撃は archer の矢だけで、味方側の弾丸状態はない。
- 描画上の複数棒人間は人数から生成した見た目であり、ゲーム状態上の独立エンティティではない。

## 変更後のルール

### 味方の独立動作

- `squadSize` を戦闘の正本にせず、`allies: Ally[]` の生存要素数から人数を導出する。
- 各味方は固有ID、HP、位置、射撃クールダウン、現在の対象敵IDを持つ。
- 射撃可能になった味方は、生存敵のうち自分から最も近い敵を選ぶ。同距離ならIDが小さい敵を選び、結果を決定論的にする。
- 射程内に敵がいない場合は射撃しない。対象が死亡または射程外になった場合は次の射撃時に再索敵する。
- 複数の味方が同じ敵を狙うことを許可する。発射後に対象が消えた弾丸は削除し、別の敵へ自動転送しない。
- 弾丸生成で味方のHPや人数を減らさない。生存している限り、同じ味方が射撃サイクルを繰り返して複数の敵を倒せる。

### 敵の独立動作

- `Enemy.count` を廃止し、画面上の敵棒人間1体を `Enemy` 1要素として保持する。
- 複数の敵が同時に存在でき、それぞれが独自の位置、HP、移動速度、対象味方ID、攻撃クールダウンを持つ。
- 各敵は最も近い生存味方を選び、その味方へ向かって独立に移動する。同距離なら味方IDで決定する。
- 対象味方が消えた場合は次の tick で再索敵する。味方が0体なら移動・攻撃を停止し game over とする。
- 通常敵は近接型、archer は遠距離型として既存のステージ進行を維持する。ボスも単一の独立エンティティとして扱うが、複数HPと専用サイズを持てる。
- 敵同士は互いを通過してよい。必要なら描画上の完全な重なりだけを避ける軽い分離を行うが、戦闘判定には使わない。

### 射撃とダメージ

- 味方の弾丸は `sourceAllyId` と `targetEnemyId` を持ち、対象の現在位置へ進む。
- 敵の遠距離攻撃は `sourceEnemyId` と `targetAllyId` を持ち、対象の現在位置へ進む。
- 命中時に対象のHPから弾丸の `damage` を引き、HPが0以下になった個体だけを削除する。
- 通常の味方と通常敵は初期値1 HPとし、1回の有効な被弾で消える。ボスなど複数HPの個体はHPが尽きるまで残る。
- 同じ tick に複数弾が命中してもHPを0未満として処理せず、撃破スコアは個体ごとに1回だけ加算する。
- 攻撃元が発射後に消えても、既に存在する弾丸は対象が生存している限り飛び続ける。

### 敵味方の接触

- 敵と味方の当たり判定が重なった場合、その敵1体と味方1体を同時に削除する。残りHPや種類にかかわらず相互消滅を優先する。
- 1 tick で1個体を複数の接触へ使用しない。候補を距離、味方ID、敵IDの順で並べ、まだ未使用の組み合わせだけを確定する。
- 接触解決は移動後、弾丸命中後に行う。先に弾丸で倒された個体は接触へ参加しない。
- 接触撃破にも個体単位のスコアを1回だけ加算する。

### ゲートと人数

- HUD の `Squad` は `allies.length` を表示する。
- `+N` は新しい味方エンティティをN体生成する。
- `-N` は決定論的な順序でN体を削除する。
- `xN` / `*N` は現在の味方を複製して目標人数まで増やす。
- `/N` は従来どおり切り捨てた目標人数まで味方を削除する。
- 全操作で最大人数を維持し、味方が0体になった時点で game over とする。

## 座標と編成

- `playerX` は部隊全体の操作中心として残す。
- 各味方の実座標は、操作中心と `formationX` / `formationY` から毎 tick 更新する。
- 味方の追加・削除後に編成オフセットを再計算し、ID順で安定した配置にする。
- 敵の索敵と弾丸の発射位置には、人数から作った描画座標ではなく各個体の実座標を使う。
- Canvas描画も同じ個体座標を使い、ゲーム判定と見た目のずれをなくす。

## 1 tick の処理順序

1. 経過時間、距離、ステージ、各個体のクールダウンを更新する。
2. スポーン条件に従い、個別の敵エンティティを生成する。
3. 味方編成の各実座標を更新する。
4. 各敵が対象味方を選び、独立して移動・遠距離攻撃する。
5. 各味方が対象敵を選び、射撃可能なら個別に弾丸を生成する。
6. 味方弾丸と敵弾を移動し、対象ごとのダメージと撃破を解決する。
7. 生存中の敵味方だけで接触候補を作り、一対一の相互消滅を解決する。
8. ゲート接触を解決し、味方エンティティを増減して編成を再計算する。
9. スコア、HUD用人数、game over を確定する。

この順序により、弾丸で先に倒された個体が同じ tick で接触ダメージも発生させる二重解決を防ぐ。

## 自動テスト計画

`src/tests.js` に少なくとも次を追加・更新する。

- 初期人数と `allies.length`、HUD用人数が一致する。
- ゲートの加算、減算、乗算、除算が個体配列へ正しく反映され、IDが重複しない。
- 複数の味方がそれぞれ最寄りの敵を選び、同距離ではID順に決まる。
- 味方が射撃してもHPと味方数が減らない。
- 同じ味方が最初の敵を倒した後に再索敵し、別の敵も倒せる。
- 対象が消えた味方弾丸は削除され、別の敵へ誤命中しない。
- 複数敵が異なる最寄り味方へ向かって独立移動する。
- archerの敵弾が指定した味方へ命中し、その味方だけにダメージを与える。
- 通常味方は有効な1ダメージで消え、他の味方は残る。
- 接触した敵味方1組だけが相互消滅する。
- 同じ個体が同一 tick の複数接触へ使われない。
- 弾丸で先に倒された敵は同じ tick の接触へ参加しない。
- 過剰ダメージや同時命中でも個体ごとの撃破スコアが重複しない。
- ボスは複数HPを維持しつつ、直接接触時は対象味方と相互消滅する。
- reset後は個体、弾丸、対象、クールダウン、IDが初期状態へ戻る。
- 最後の味方が被弾または接触で消えると game over になる。

## ブラウザ検証

- 1280x720で複数の味方が別々の敵へ発砲し、弾道と対象が一致する。
- 味方1体の状態で複数の敵を連続して射撃撃破できる。
- 複数敵が同時に異なる味方へ動き、全員が同じ経路に固定されない。
- 敵弾が命中した味方だけが消え、HUD人数が1減る。
- 接触した敵味方の組だけが消え、近くにいる未接触個体は残る。
- ゲート通過後に味方の個体数と編成描画が一致する。
- archer、ボス、game over、Restartが個体モデルで正常に動く。
- 最大人数と多数の敵・弾丸が存在してもフレーム落ちや無制限な弾丸増加がない。
- 390x844で横スクロール、HUD崩れ、タッチ操作の退行がない。

## 受け入れ条件

- 各味方棒人間が独自に敵を選び、自動射撃する。
- 味方1体が消費されずに複数の敵を順次倒せる。
- 各敵棒人間が独自に対象を選び、移動・攻撃する。
- 敵の攻撃が命中した味方個体だけがダメージを受ける。
- 直接衝突した敵味方は一対一で同時に消える。
- 複数の同時命中・接触でも二重削除や二重スコアが起きない。
- ゲート、archer、ボス、スコア、game over、Restartが個体モデルと整合する。
- `npm test` とブラウザ検証が成功する。

- プレイヤーが個々の味方を直接操作する機能。

- サウンドや新規画像アセット。

## リスクと前提

- 「敵キャラはそれぞれ独立動作」は、複数の敵が同時に存在し、各敵が個別の対象・移動・攻撃状態を持つ意味とする。
- 「敵の攻撃」は既存 archer の遠距離攻撃を最低要件とし、通常敵の直接接触は攻撃ダメージではなく相互消滅で処理する。
- 「ぶつかっても消える」は残りHPに関係なく、接触した敵味方1組が同時に消える意味とする。
- 通常の棒人間は1 HPとする。将来HPを増やしても被弾処理を再利用できるデータモデルにする。
- 索敵距離、射撃間隔、弾速、敵速度、最大同時敵数は定数化し、プレイテストで調整する。
- 個体数と弾丸数が増えるため、単純な全組み合わせ判定が負荷になる場合は空間分割を後続最適化として導入する。
