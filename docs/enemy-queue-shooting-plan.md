# 敵の単体追跡・味方射撃 実装プラン

## 目的

現在の「敵グループが直進し、接触時に味方人数と相殺する」戦闘を、次のルールへ変更する。

- 敵は同時に1体だけフィールド上を進む。
- 敵はプレイヤー部隊の現在位置へ向かって移動し、横を通り過ぎない。
- 味方は敵へ自動で銃弾を撃ち、命中した銃弾で敵を倒す。
- 射撃しても味方は消費されないため、味方1体でも複数の敵を順番に倒せる。

この文書は実装方針を定義する。ゲーム本体の変更は、別途明示的に実装を依頼されたときに行う。

## 現状

- `state.enemies` は複数の敵グループを保持でき、各グループは `count` を持つ。
- 敵は固定された `x` のまま画面下方向へ進み、プレイヤーと横位置が合わなければ通過できる。
- 接触時は `min(squadSize, enemy.count)` を双方から引く一対一相殺で戦闘を解決する。
- 敵の射撃は後半の `archer` が生成する `arrows` のみで、味方側の弾丸状態はない。
- 描画は敵の `count` を群衆として表示し、README も「味方は射撃しない」と説明している。

## 変更後のゲームルール

### 敵の進行

- `state.enemies` に存在できる生存中の通常敵またはボスは最大1体とする。
- 次の敵は、現在の敵が倒された後にスポーンクールダウンを経て生成する。ステージ4以降でボスが未生成なら、通常敵よりボス生成を優先する。
- 敵は毎 tick、`x` をプレイヤーの `playerX` へ近づけながら `y` を前進させる。横移動量は delta time に基づいて制限し、フレームレートに依存させない。
- 敵はプレイヤー前面の接触位置で停止する。接触判定を飛び越える大きな delta time でも、プレイヤーより後方へ抜けないよう位置を clamp する。
- 接触した通常敵は味方を1体倒して消滅する。ボスは接触攻撃クールダウンごとに味方を1体減らし、HP が残る間は接触位置に留まる。
- 味方人数が0になった場合は従来どおり game over とする。

### 味方の射撃

- 状態へ `bullets` と `playerFireCooldown` を追加する。
- 生存中の敵がいる場合だけ、味方は一定間隔で自動射撃する。
- 1回の射撃で生成する弾数は `min(PLAYER_SALVO_LIMIT, squadSize)` とする。`squadSize` が0なら射撃せず、1以上なら最低1発を生成する。味方人数は弾丸生成によって減らさない。
- 各弾丸は生成時の対象敵IDを持ち、毎 tick で現在の対象位置へ向かう。対象が消滅した弾丸は削除し、次の敵へ自動転送しない。
- 通常敵は原則1 HP、ボスは複数 HP とする。弾丸1発は1ダメージを与える。
- 敵撃破時に既存の種別別スコアを加算し、同じ敵へ同一 tick で複数弾が命中しても撃破スコアは1回だけ加算する。

### 表示

- 通常敵は群衆ではなく単体として描画し、ボスのみ大きな単体キャラクターとして描画する。
- HP が2以上の敵にはHP表示を出す。通常敵の不要な人数バブルは削除する。
- 味方から対象敵へ飛ぶ銃弾を描画し、弾道と命中対象が視覚的に一致するようにする。
- HUD の `Squad` は戦闘リソースではなく残存する味方人数として引き続き表示する。
- README のゲーム説明を新しい戦闘ルールへ更新する。

## 状態モデル案

```text
GameState {
  squadSize: number
  playerFireCooldown: number
  enemies: Enemy[]       // 長さは常に 0 または 1
  bullets: Bullet[]
  arrows: Arrow[]        // archer の既存攻撃を維持
}

Enemy {
  id: number
  type: "grunt" | "archer" | "boss"
  x: number
  y: number
  hp: number
  maxHp: number
  speed: number
  trackingSpeed: number
  contactCooldown: number
  fireCooldown: number
}

Bullet {
  id: number
  targetId: number
  x: number
  y: number
  speed: number
  damage: 1
}
```

`Enemy.count` は `hp` / `maxHp` に置き換える。通常敵を単体として扱うため、接触相殺用の人数情報は保持しない。

## 処理順序

1 tick の更新順序を次に固定する。

1. 経過時間、距離、ステージ、各クールダウンを更新する。
2. 生存中の敵がいない場合だけ次の敵を生成する。ボス出現条件を満たす場合はボスを優先する。
3. 敵をプレイヤーへ追跡移動させ、接触位置で停止させる。
4. 味方の射撃条件を評価し、対象ID付き弾丸を生成する。
5. 弾丸を対象へ移動し、命中ダメージと撃破スコアを解決する。
6. 生存している archer の射撃と矢の命中を解決する。
7. 生存している敵の接触攻撃を解決する。
8. ゲート接触を解決する。

弾丸命中を敵の接触攻撃より先に処理することで、接触直前に倒した敵から不自然なダメージを受けないようにする。

## 自動テスト計画

`src/tests.js` に少なくとも次を追加・更新する。

- 敵がいないときだけ1体生成される。
- 生存中の敵がいる間はスポーンクールダウンが経過しても2体目が生成されない。
- 敵の `x` が左右どちらからでも `playerX` へ近づく。
- 大きな delta time でも敵の `y` がプレイヤー接触位置を越えない。
- 味方が0人または game over のときは弾丸を生成しない。
- 味方1人でも、射撃後に `squadSize` が減らない。
- 味方1人が敵を倒した後、次に生成された別の敵も倒せる。
- 対象敵が消滅した弾丸は安全に削除される。
- 同一 tick の過剰ダメージで撃破スコアが重複しない。
- 通常敵の接触で味方が1人減り、敵は通過せず消滅する。
- ボスは複数発を受けるまで残り、接触位置から通過しない。
- archer の矢、ゲート、reset、game over の既存動作が維持される。

## ブラウザ検証

- 味方が複数いると salvo が増えるが、描画負荷と弾丸数が設定上限内に収まる。
- archer とボスの攻撃・HP表示が正常に機能する。
- 390x844 で横スクロールやHUD崩れがない。
- Restart 後に敵、弾丸、クールダウンが初期状態へ戻る。

## 受け入れ条件

- フィールド上の生存中の敵は常に最大1体である。
- 敵はプレイヤーの横位置を追跡し、接触位置より後ろへ通過しない。
- 味方は敵がいる間、自動で銃弾を発射する。
- 銃弾を撃っても味方人数は減らない。
- 味方1体で複数の敵を順番に倒せる。
- 通常敵、archer、ボス、ゲート、game over、restart が新しい戦闘モデルと矛盾なく動作する。
- `npm test` とブラウザ検証が成功する。

- サウンドや新規画像アセットの追加。

## リスクと前提

- 「一体ずつ」は、画面上で動く生存敵を最大1体にする意味として扱う。複数の敵を同時表示して順番に動かす仕様にはしない。
- 「通り過ぎない」は、横追跡と接触位置 clamp の両方で保証する。
- 通常敵が接触した場合の結果は「味方1体と敵1体が失われる」とする。これにより射撃に失敗した際のペナルティを残す。
- 既存 archer の敵側射撃は維持する。味方射撃への変更だけを理由に削除しない。
- salvo 上限、発射間隔、敵HP、移動速度はプレイテストで調整可能な定数として定義する。
